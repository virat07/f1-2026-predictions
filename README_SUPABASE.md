# Supabase Setup for F1 2026 Predictions

To make the community leaderboard and predictions work globally, follow these steps:

## 1. Create a Supabase Project
Go to [supabase.com](https://supabase.com) and create a new project.

## 2. Configure Client
Update `src/utils/supabaseClient.js` with your **Project URL** and **Anon Key** found in the Supabase Dashboard under `Project Settings > API`.

## 3. Create Tables
Run the following SQL in the **SQL Editor** tab of your Supabase Dashboard:

```sql
-- Create the leaderboard table
CREATE TABLE leaderboard (
  username TEXT PRIMARY KEY,
  score INTEGER DEFAULT 0,
  joined_at TIMESTAMPTZ DEFAULT NOW()
);

-- Create the predictions table
CREATE TABLE predictions (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  username TEXT REFERENCES leaderboard(username),
  round INTEGER,
  predictions JSONB, -- stores {qualy: "...", sprint: "...", race: "..."}
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(username, round)
);

-- Create a results table to store actual outcomes
CREATE TABLE actual_results (
  round INTEGER PRIMARY KEY,
  qualy_winner TEXT,
  sprint_winner TEXT,
  race_winner TEXT,
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Enable Row Level Security (RLS)
ALTER TABLE leaderboard ENABLE ROW LEVEL SECURITY;
ALTER TABLE predictions ENABLE ROW LEVEL SECURITY;
ALTER TABLE actual_results ENABLE ROW LEVEL SECURITY;

-- Allow public read/write for demo purposes (In production, use Auth)
CREATE POLICY "Public Read Leaderboard" ON leaderboard FOR SELECT USING (true);
CREATE POLICY "Public Insert Leaderboard" ON leaderboard FOR INSERT WITH CHECK (true);
CREATE POLICY "Public Update Leaderboard" ON leaderboard FOR UPDATE USING (true);

CREATE POLICY "Public Read Predictions" ON predictions FOR SELECT USING (true);
CREATE POLICY "Public Insert Predictions" ON predictions FOR INSERT WITH CHECK (true);
CREATE POLICY "Public Update Predictions" ON predictions FOR UPDATE USING (true);

CREATE POLICY "Public Read Results" ON actual_results FOR SELECT USING (true);
CREATE POLICY "Public Insert Results" ON actual_results FOR INSERT WITH CHECK (true);
CREATE POLICY "Public Update Results" ON actual_results FOR UPDATE USING (true);

-- FUNCTION: Automatically calculate scores when results are updated
-- Points: Qualy (5), Sprint (10), Race (25)
CREATE OR REPLACE FUNCTION update_user_scores()
RETURNS TRIGGER AS $$
BEGIN
  -- Update scores for all users who made predictions for this round
  UPDATE leaderboard l
  SET score = (
    SELECT COALESCE(SUM(
      (CASE WHEN (p.predictions->>'qualy') = NEW.qualy_winner THEN 5 ELSE 0 END) +
      (CASE WHEN (p.predictions->>'sprint') = NEW.sprint_winner THEN 10 ELSE 0 END) +
      (CASE WHEN (p.predictions->>'race') = NEW.race_winner THEN 25 ELSE 0 END)
    ), 0)
    FROM predictions p
    WHERE p.username = l.username
  );
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

-- TRIGGER: Run the score update whenever a new result is saved
CREATE TRIGGER on_result_posted
AFTER INSERT OR UPDATE ON actual_results
FOR EACH ROW
EXECUTE FUNCTION update_user_scores();
```

## 4. Deploy
Once configured, your Vercel deployment will automatically use Supabase to sync data across all users!
